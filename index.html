User Flow Diagram

1. Account Creation
- User fills account creation form
- System creates root user account and organization
- User is redirected to Central Dashboard
↓
2. Staff Access Control
- Root user clicks on "Staff Access Control"
- Root user creates new staff member account
- Root user assigns permissions to staff member
- System logs activity
- System updates staff permissions database
↓
3. Staff Login
- User enters staff-id and password
- System authenticates user
- System logs activity
- System retrieves user permissions
- User is redirected to Central Dashboard with only tabs user has access too
↓
4. Inventory Management
- User with access clicks on "Inventory Management" (IM)
- User is taken to Inventory Management page
- User views inventory list
- User adds/updates inventory quantity
- User clicks "Save"
- System verifies permissions
- System updates inventory database (DB)
- System pushes update to every other user with inventory access
↓
5. Section Management
- User with access clicks on "Section Management" (SM)
- User views section list
- User creates new section
- User clicks "Save"
- System verifies permissions
- System updates section database (DB)
- User allocates inventory to section
- User clicks "Save"
- System verifies permissions
- System updates section and permission database (DB)
↓
6. Staff Access Control
- User with access clicks on "Staff Access Control" (SAC)
- User assigns section permissions to staff member
- System verifies permissions
- System updates staff permissions, section and user database (DB)
- System logs activity (AL)
↓
6. Activity Logging
- User views activity logs
- User filters logs by date or staff member
- System displays filtered logs

User Interactions

  • Clicks
  • Form submissions (login/signup, inventory updates, section creations)
  • Page navigations

System Responses

  • Authenticates user
  • Updates databases (inventory, sections, staff permissions)
  • Logs activities
  • Displays updated data
